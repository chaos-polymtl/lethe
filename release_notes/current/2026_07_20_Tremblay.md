## [Master] - 2026/07/17

### Added

- MAJOR Implements the ability to use a different rotation axis than the z-axis when using the mortar method. Also makes it so that the rotation axis doesn't need to pass through the origin. The code is still limited to a rotation axis parallel to the main x, y or z axis. [#2016](https://github.com/chaos-polymtl/lethe/pull/2016)

### Fixed

- MINOR Fixes a bug when calculating the normal vectors at the mortar interface when the rotation axis is not passing through the origin. The code would still assume the center of rotation to be at (0,0), which would affect the results. [#2016](https://github.com/chaos-polymtl/lethe/pull/2016)
