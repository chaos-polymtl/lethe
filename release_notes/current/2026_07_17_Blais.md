## [Master] - 2026/07/17

### Added

- MAJOR A new `stabilization parameter` option (`tezduyar|metric_tensor`) is added to the `stabilization` subsection to select how the Navier-Stokes stabilization parameter tau is defined. The default `tezduyar` reproduces the previous isotropic definition based on a single scalar element size. The new `metric_tensor` definition builds tau from the anisotropic element covariant metric tensor `G = J^{-T} J^{-1}` (Bazilevs et al. variational multiscale approach), which retains the directional size information of the element and is therefore more robust for deformed, stretched or high-aspect-ratio cells. To avoid under-stabilizing the pressure of equal-order elements on high-aspect-ratio cells (where the anisotropic tau collapses), the `metric_tensor` definition is applied only to the SUPG and LSIC terms while the PSPG term always keeps the isotropic definition. The metric tensor and its associated vector are scaled with the polynomial degree so that the definition remains consistent and robust for higher-order elements. This feature is currently only supported by the matrix-free solver (`lethe-fluid-matrix-free`). [#XXXX](https://github.com/chaos-polymtl/lethe/pull/XXXX)
