## [Master] - 2026/07/16

### Fixed

- MAJOR Fixed a bug that prevented `lethe-fluid-particles-matrix-free` simulations from restarting consistently. When restoring a checkpoint, the fluid solution vectors (`present_solution` and `previous_solutions`) were deserialized with valid locally owned entries but stale ghost values. On the first restarted step, the particle-fluid force/drag projection evaluates the fluid solution at the particle locations *before* the first Newton solve (which is what normally refreshes the ghosts), so particles located in cells touching a subdomain boundary read incorrect ghost values. This produced drag and momentum-transfer-coefficient fields that were off by several percent, and consequently a different fluid residual and solution on the restarted step compared to an uninterrupted run. The ghost values of the restored fluid vectors are now refreshed in `read_checkpoint`, restoring the invariant (assumed elsewhere in the solver) that these vectors are always ghosted after a solve. Restarted CFD-DEM matrix-free simulations now reproduce an uninterrupted run down to particle-ordering round-off. [#2060](https://github.com/chaos-polymtl/lethe/pull/2060)

- MINOR Fixed the void fraction update in the load balancing step of `lethe-fluid-particles-matrix-free`. The Trilinos void fraction vector was converted from the deal.II `void_fraction_solution` before the freshly interpolated field was stored into it (picking up the zeroed, post-`setup_dofs` content instead), and a stray comma operator joined two assignments. The interpolated field is now stored before the conversion. [#2060](https://github.com/chaos-polymtl/lethe/pull/2060)
