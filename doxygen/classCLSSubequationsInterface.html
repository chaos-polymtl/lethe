◆ subequations_solutions_validity

template<int dim>
std::map<CLSSubequationsID, bool> CLSSubequationsInterface< dim >::subequations_solutions_validity
private

Booleans indicating if the subequation has been solved with the set CLS solution vector. The CLS solution vector is set with CLSSubequationsInterface<dim>::set_cls_solution_and_dof_handler.

During initialization (CLSSubequationsInterface<dim>::initialize_subequations), all boolean are set to false.

We start by setting a CLS solution and DOFHandler (CLSSubequationsInterface<dim>::set_cls_solution_and_dof_handler), and then start solving equations making sure that dependencies are met (CLSPhaseGradientProjection<dim>::check_dependencies_validity, CLSCurvatureProjection<dim>::check_dependencies_validity, and CLSPDEBasedInterfaceReinitialization<dim>::check_dependencies_validity). For instance, to solve the curvature projection subequation, the phase gradient projection subequation has to be solved before as the projected phase gradient is used in the curvature projection subequation.

A subequation in the validity map is to true only when it is solved, and it remains valid until a new CLS solution and/or DOFHandler are set through CLSSubequationsInterface<dim>::set_cls_solution_and_dof_handler.

This validation mechanism allows, to check if:

  • A new CLS solution has been set;
  • All dependencies of the subequation have been solved;
  • The current solution of the subequation is valid, and avoid unnecessarily solving it once more.
Note
CLSSubequationsInterface<dim>::set_cls_solution_and_dof_handler also resets the validity map values to false.
