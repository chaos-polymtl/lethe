Turn regions of the mesh where the material_id>0 into a solid block by injecting velocity and pressure DOFs into the constraints.

It is achieved by imposing \(\mathbf{u}=0\) within the cells which have a material_id>0. In addition, solid cells which are not connected to the fluid by any means also get a pressure Dirichlet boundary condition which fixes the pressure to 0. It ensures that the linear system is well-posed. Right now, this routine only supports the usage of 1 solid domain, but eventually it could be extended to more than one. By default, the fluid domain is assumed to have a material_id=0 and the rest of the domains have a material_id>0.

Parameters
[in]dof_handlerDoFHandler for which the solid region is established. In the majority of cases this is the DoFHandler that is a member of this class, but if a multigrid preconditioner, constraints must be built at every level. In this case, this DoFHandler can be the DoFHandler at any of the multigrid level.
[in]locally_owned_dofsThe locally owned DoFs for the dof_handler. Used to filter pressure DoFs that should receive a Dirichlet constraint when their cell is fully disconnected from the fluid.
[in]non_zero_constraintsIf this parameter is true, it indicates that the constraints used for the solid domain are the non-zero constraints. If this is set to false, the constraints being used are zero constraints.
[in]constraintsSet of constraints on which the solid domain constraint is applied.

Multigrid-level variant of establish_solid_domain for the local smoothing multigrid (LSMG) preconditioner.

Unlike the active-cell version above, this overload walks the cells of a specific multigrid level using mg_cell_iterators_on_level and uses get_mg_dof_indices, so the DOF indices it adds to the constraint object match the multigrid-level IndexSet with which constraints was reinit'd (i.e. locally_owned_mg_dofs(level) and the corresponding locally relevant level DoFs). Using the active-cell version on an LSMG level constraint object triggers the local_lines.is_element(line_n) assertion inside AffineConstraints::add_line because active and MG DoF numberings are unrelated.

The material_id is inherited from active cells by their parents on coarser levels, so the solid/fluid classification is well defined on MG levels.

Parameters
[in]dof_handlerDoFHandler distributed with distribute_mg_dofs.
[in]levelMultigrid level on which constraints are built.
[in]locally_owned_mg_dofsThe locally owned MG DoFs on level (i.e. dof_handler.locally_owned_mg_dofs(level)). Used to filter pressure DoFs that should receive a Dirichlet constraint when their cell is fully disconnected from the fluid.
[in]non_zero_constraintsIf true, set inhomogeneous (zero-valued) constraints; otherwise add homogeneous constraints.
[in,out]constraintsLevel constraint object to populate.
